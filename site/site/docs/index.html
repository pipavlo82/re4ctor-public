⟵ Re4ctoR

Re4ctoR API v1

High-entropy randomness API delivered via Cloudflare edge. Core latency <1 ms, typical global SaaS latency ~20–30 ms.

Base URL: https://api.re4ctor.com Auth: X-API-Key Demo key: demo

Authentication

All requests must include an API key header: 

X-API-Key: demo

The public demo key is rate-limited and not suitable for production. For dedicated keys contact us at security@re4ctor.com.

Endpoints

GET /v1/health

Simple health check for the SaaS gateway.

curl https://api.re4ctor.com/health
GET /v1/random

Core high-entropy RNG. Returns unsigned randomness from the Re4ctoR core (suitable when you already terminate TLS and do not need signatures).

Query parameters:

  • n — number of bytes (1–4096)
  • fmt — bin (default) or hex
curl "https://api.re4ctor.com/v1/random?n=8&fmt=hex" \
  -H "X-API-Key: demo"
GET /v1/random_pq

Randomness with post-quantum signature attached (PQ-only or hybrid depending on plan). Useful when you need verifiable randomness off-chain or inside secure enclaves.

Query parameters:

  • n — number of bytes
  • sig — ml-dsa, dilithium (roadmap)
curl "https://api.re4ctor.com/v1/random_pq?n=32&sig=ml-dsa" \
  -H "X-API-Key: demo"
GET /v1/random-dual

Dual-signature endpoint: classical ECDSA(secp256k1) + post-quantum signature in a single JSON payload. Designed for blockchains and long-lived data. 

curl "https://api.re4ctor.com/v1/random-dual?n=32&sig=ecdsa" \
  -H "X-API-Key: demo"

Code examples

Python

import requests

BASE = "https://api.re4ctor.com"
headers = {"X-API-Key": "demo"}

r = requests.get(f"{BASE}/v1/random", params={"n": 8, "fmt": "hex"},
                 headers=headers, timeout=3.0)
r.raise_for_status()
print("Random hex:", r.text.strip())

Node.js

import fetch from "node-fetch";

const BASE = "https://api.re4ctor.com";

const res = await fetch(
  `${BASE}/v1/random?n=8&fmt=hex`,
  { headers: { "X-API-Key": "demo" } }
);
const body = await res.text();
console.log("Random hex:", body.trim());

Limits & SLA

Public demo tier:

  • Up to 10 requests / second
  • Up to 4 KB randomness per request
  • No uptime guarantees, best-effort only

Dedicated and enterprise plans include higher rate limits, custom SLAs, self-hosted deployment and air-gapped / defense editions.
