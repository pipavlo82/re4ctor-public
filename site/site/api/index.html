API Documentation

Re4ctoR exposes a hardened HTTP gateway in front of the Core RNG and VRF node. All public calls (except health) require X-API-Key.

Base URL

Production gateway: https://api.re4ctor.com

Authentication

Header: X-API-Key: demo (dev / demo plan)

Endpoints

GET /v1/health

Lightweight health check of the gateway.

curl -s "https://api.re4ctor.com/v1/health"
GET /v1/random

High-entropy random bytes proxied from RE4CTOR Core RNG.

QueryTypeDescription
nintNumber of bytes to return (e.g. 16, 32).
fmtstringhex (default) or raw.
curl -s -H "X-API-Key: demo" \
  "https://api.re4ctor.com/v1/random?n=16&fmt=hex"
GET /v1/vrf

Dual-signed VRF-style randomness, proxied до VRF node (всередині він викликає свій /random_dual). Відповідь — JSON з ECDSA(secp256k1) + ML-DSA-65.

QueryTypeDescription
sig string Signature scheme. For now: ecdsa (dual ECDSA + ML-DSA-65). 
curl -s -H "X-API-Key: demo" \
  "https://api.re4ctor.com/v1/vrf?sig=ecdsa" | jq .
POST /v1/verify

Helper endpoint для перевірки Ethereum-style ECDSA підпису від VRF (msg_hash / v / r / s) на API боці. 

curl -s -X POST "https://api.re4ctor.com/v1/verify" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: demo" \
  -d '{
    "msg_hash": "0x...",
    "r": "0x...",
    "s": "0x...",
    "v": 27,
    "expected_signer": "0x1C901e3bd9..."
  }' | jq .
